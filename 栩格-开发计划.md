# 栩格 — AI 知识体检与复习助手 · 开发计划

> **项目名称：** 栩格
> **赛道：** 学习工作
> **口号：** 把你的知识管起来，让遗忘有据可查
> **核心闭环：** 上传笔记 → AI 分析提取知识点 → AI 出题 → 用户作答 → 即时诊断薄弱点 → 针对性补漏
> **上线目标：** 稳定可用，用户能注册使用，支持运营

## 五、TRAE 特色运用（写在方案里，体现技术创新）

### 4.1 TRAE Work 自动化知识分析流水线

笔记上传后，不是简单调一次 API 就完事。用 TRAE Work 编排一个**有状态的异步流水线**：

```
上传完成
  │
  ▼ [TRAE Work 触发]
  ├── Step 1: 文本提取（pdf-parse/mammoth）
  ├── Step 2: AI 知识点提取 + 摘要生成
  ├── Step 3: 自动分类打标签
  └── Step 4: 更新数据库状态 + 通知前端
```

对比传统方案（轮询或 WebSocket），TRAE Work 的编排可以做到：
- 每个步骤独立可追踪失败
- 支持步骤间条件分支（如大文件走不同 Prompt）
- 运行时状态可见，调试方便

### 4.2 TRAE Builder 自适应复习计划

每次复习结束后，TRAE Builder 根据以下状态自动生成下一轮复习计划：
- **当前用户答错的题** → 优先加入下一轮
- **根据遗忘曲线计算的时间戳** → 某个知识点距上次复习已超过最佳间隔
- **弱项报告中的低掌握度知识点** → 提高权重

```typescript
// TRAE Builder 生成的复习计划结构
interface ReviewPlan {
  priority: KnowledgePoint[];   // 需要立即复习的（错题+超期）
  normal: KnowledgePoint[];     // 常规轮换
  skip: KnowledgePoint[];       // 掌握度高，暂不需复习
  schedule: {                   // 未来排期
    tomorrow: string[];
    threeDays: string[];
    week: string[];
  };
}
```

### 4.3 TRAE 代码理解能力（技术类笔记差异化）

对于上传技术文档/代码笔记的用户，TRAE 的代码理解能力比通用大模型做得更好：
- 自动识别代码中的函数定义、设计模式、架构决策
- 生成"这个 API 怎么用"式的实用性题目，而非纯理论问答
- 对技术类知识点的提取粒度更精细

## 十、部署与运营规划

### 10.1 部署架构

```
用户浏览器 → 域名 (xuge.com / 服务器IP)
    │
    ▼
云服务器 (2核4G 轻量服务器)
    ├── Nginx（端口 80/443）
    │   ├── / → React 静态文件（已构建）
    │   └── /api/* → 反向代理到 localhost:3001
    │
    ├── PM2 → Express 后端（端口 3001）
    │
    └── PostgreSQL（端口 5432）
         └── 上传文件目录 /app/uploads
```

### 10.2 服务器初始化（半小时）

购买阿里云/腾讯云轻量服务器后，SSH 进去跑：

```bash
# 安装 Node.js 22
curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash -
sudo apt install -y nodejs

# 安装 PostgreSQL
sudo apt install -y postgresql postgresql-contrib

# 安装 Nginx
sudo apt install -y nginx

# 安装 PM2
sudo npm install -g pm2

# 创建数据库
sudo -u postgres psql -c "CREATE DATABASE xuge;"
sudo -u postgres psql -c "CREATE USER xuge_user WITH PASSWORD '你的密码';"
sudo -u postgres psql -c "GRANT ALL PRIVILEGES ON DATABASE xuge TO xuge_user;"
```

### 10.3 部署应用

```bash
# 拉取代码
git clone https://github.com/你的仓库/xuge.git
cd xuge/server

# 配置环境变量
cp .env.example .env
# 编辑 .env，填入 DATABASE_URL=postgresql://xuge_user:密码@localhost:5432/xuge

# 安装依赖 + 初始化数据库
npm install
npx prisma migrate deploy

# PM2 启动（宕机自动重启）
pm2 start index.js --name xuge-server
pm2 save
pm2 startup   # 服务器重启后自动启动

# 构建前端
cd ../client
npm install
npm run build   # 生成 dist/ 目录，Nginx 可直接提供服务
```

### 10.4 Nginx 配置

```nginx
server {
    listen 80;
    server_name 你的域名或IP;

    root /home/xuge/client/dist;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /api/ {
        proxy_pass http://localhost:3001;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    location /uploads/ {
        alias /app/uploads/;
    }
}
```

配置 HTTPS：`sudo apt install certbot python3-certbot-nginx && sudo certbot --nginx`
